## Running the app

Run this from the project root (the folder containing `app/` and `src/`):

```powershell
.venv\Scripts\python.exe -m uvicorn app.main:app --reload --port 8000
```

Then open **http://localhost:8000** in your browser.

> No activation step needed — call the venv Python directly to avoid PowerShell execution-policy issues.

## Running tests

```powershell
python -m pytest tests\ -v
```

## What was built

**Computation layer** (`src/fetal_brain_mri/`):
- All 15 biometric parameters registered with verbatim SPEC §7.3 coefficients
- 14 base DDx triggers + 5 combined-pattern cards + Chiari II Mahalanobis discriminator
- Extended band classification: `<3rd` / `<5th` / `normal` / `>95th` / `>97th`
- `evaluate_case()` API: takes all measurements → returns z-scores, bands, DDx cards, report text

**Web app** (`app/`):
- FastAPI + Jinja2 + HTMX — updates live as the user types (400 ms debounce)
- Anatomically grouped measurement worksheet with per-row z-score, percentile, band, and source disclosure
- DDx accordion cards with likelihoods, next steps, and citations
- Copy-to-clipboard report for PowerScribe

## Clinical caveats (must communicate before deployment)

1. **Third ventricle z-score** — approximation only; use the 3.5 mm threshold, not the z-score, for clinical decisions. Needs Hertzberg 1997 Table 1 transcription.
2. **Dovjak 2021 coefficients** — need eyes-on-paper verification against Wiley OBGYN Table 1 (doi:10.1002/uog.22162).
3. **TDPF / CSA coefficients** — OLS-derived from Woitek 2014 Table 3; must be re-verified against the original paper (PMC4231033).
4. **Chiari II card** — research-mode only; trained on a single 44-case cohort; requires local calibration before clinical use.
5. **Cross-modality warning** — Birnbaum 2018 (third ventricle) is an ultrasound source applied to MRI measurements; the report flags this.
